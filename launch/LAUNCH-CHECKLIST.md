# Checklist de lancement — de zéro à en vente

Tout est prêt à copier (`gumroad-listings.md`, `social-posts.md`). Voici l'ordre d'exécution.
Coche au fur et à mesure. Objectif : **en vente en 1 journée**, puis diffusion régulière.

## Jour 0 — Mise en vente (~1 h 30)

- [ ] Créer un compte **Gumroad** (ou Lemon Squeezy). C'est eux qui encaissent.
- [ ] Pour chaque produit : zipper son dossier (sans `.git`).
  - Aurora = racine du repo · Nova/Mentor/Onelink = leur dossier dans `templates/`
- [ ] Créer les 5 produits avec les textes de `gumroad-listings.md` (4 templates + bundle).
- [ ] Faire 3–5 captures propres par produit (clair + sombre + mobile). **C'est ce qui vend.**
- [ ] Fixer les prix : Onelink 12 € · les 3 landings 29–34 € · bundle 59 €.
- [ ] Activer la garantie 14 jours dans les réglages Gumroad.
- [ ] Tester un achat test (mode preview) pour vérifier le fichier livré.

## Jour 1 — Premier canal (le plus fort pour ta niche)

- [ ] Publier **1** post là où vit ta cible principale (voir `social-posts.md`).
- [ ] Répondre à **chaque** commentaire dans les 2 h.
- [ ] Noter le nombre de visites / ventes.

## Jours 2–7 — Un canal par jour

- [ ] Jour 2 : Reddit (sub adapté au produit).
- [ ] Jour 3 : X/Twitter thread #buildinpublic.
- [ ] Jour 4 : LinkedIn (surtout pour Mentor).
- [ ] Jour 5 : TikTok/Instagram (surtout pour Onelink).
- [ ] Jour 6 : Product Hunt (prépare la veille, lance à 00:01 PST).
- [ ] Jour 7 : bilan — quel canal a converti ? Double dessus.

## Semaines 2–8 — Régularité

- [ ] **1 action de diffusion / jour** (post, réponse, DM à une agence).
- [ ] Ajouter les avis clients sur les fiches produit (preuve sociale = plus de ventes).
- [ ] Proposer un code promo -20 % ponctuel pour créer de l'urgence.
- [ ] Recycler les meilleurs posts qui ont marché.
